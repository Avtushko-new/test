Bug Report: Метод trim() некорректно обрабатывает пробельные символы
1. Основная информация
Компонент: StringUtils.trim()

Критичность: High

Приоритет: High

Тип дефекта: Функциональный

2. Описание проблемы
Метод trim() не удаляет:

Символы табуляции (\t)

Символы переноса строки (\n)

Комбинации пробелов, табуляции и переносов строки

Ожидаемое поведение:
Метод должен удалять все пробельные символы в начале строки, включая:

Обычные пробелы ( )

Табуляцию (\t)

Переносы строк (\n)

Любые их комбинации

3. Шаги для воспроизведения
Тест 1:

python
input_str = "\t\n текст" 
expected = "текст"
assert trim(input_str) == expected  # Фактический результат: "\t\n текст"
Тест 2:

python
input_str = "  \t\n  "  
expected = ""
assert trim(input_str) == expected  # Фактический результат: "\t\n  "
4. Фактические и ожидаемые результаты
Входные данные	Ожидаемый результат	Фактический результат
"\t\n текст"	"текст"	"\t\n текст"
" \t\n "	""	"\t\n "
5. Причина ошибки
Метод trim() использует только проверку на обычные пробелы (whitespace = " "), но не учитывает:

\t (табуляцию)

\n (перенос строки)
